# 하이콰 (HAYQUA) - 음식점 메뉴 웹사이트

Next.js로 제작된 음식점 메뉴 웹사이트입니다. 디자이너와 개발자가 효율적으로 협업할 수 있도록 설계되었습니다.

## 🎯 주요 특징

- **마크다운 데이터베이스**: .md 파일로 메뉴를 쉽게 관리
- **숫자 기반 카테고리**: 간단한 숫자로 카테고리 입력 (1-8)
- **완전한 관심사 분리**: 카테고리와 메뉴를 별도 파일로 관리
- **디자인 시스템**: 재사용 가능한 UI 컴포넌트
- **브랜드 테마**: 브랜드 컬러 시스템 (#1a2332 네이비, #f4a460 골드)
- **타입 안정성**: TypeScript로 타입 안정성 보장
- **반응형 디자인**: 모든 디바이스 최적화

## 🍜 메뉴 관리하기 (누구나 가능!)

### 📂 카테고리 매핑 이해하기

`menu-database.md` 상단에 카테고리 매핑이 있습니다:

```markdown
## 카테고리 ID 매핑

1: pho (칼국수)
2: friedNoodles (볶음면)
3: friedRice (볶음밥)
4: special (스페셜메뉴)
5: side (애피타이져)
6: hotfood (안주류)
7: drink (음료)
8: alcohol (주류)
```

메뉴 추가 시 `**카테고리**` 필드에 **숫자만 입력**하면 됩니다!

### 1️⃣ 카테고리 관리 (`data/category-database.md`)

카테고리 정보와 설명을 관리합니다:

```markdown
## 새 카테고리 이름
- **ID**: newCategory
- **영문명**: New Category
- **설명**: 카테고리 설명
```

**✨ 순서 변경:** 카테고리를 위아래로 이동하면 웹사이트의 표시 순서도 변경됩니다!

**⚠️ 카테고리 추가 시:**
1. `category-database.md`에 새 카테고리 추가
2. `menu-database.md` 상단의 매핑에 숫자 추가 (예: `9: newCategory`)

### 2️⃣ 메뉴 추가 (`data/menu-database.md`)

메뉴 정보만 입력합니다. **카테고리는 숫자로!**

```markdown
### 새로운 메뉴
- **ID**: 58
- **카테고리**: 1  👈 숫자로 입력! (1=칼국수)
- **영문명**: New Menu
- **가격**: 12,000원
- **이미지**: /images/new.jpeg
- **추천**: ⭐
- **BEST**: ⭐
- **옵션**:
  - 작은 사이즈: 8,000원
  - 큰 사이즈: 12,000원
- **비고**: 특별 할인 중
```

### ✏️ 3️⃣ 메뉴 수정

기존 메뉴의 내용을 수정하세요:

```markdown
### 점보계란말이
- **ID**: 1
- **카테고리**: 6  👈 숫자로! (6=안주류)
- **영문명**: Jumbo Egg
- **가격**: 16,000원  👈 가격 변경
- **추천**: ⭐        👈 추천 메뉴로 설정
- **이미지**: /images/egg.jpeg  👈 이미지 추가
```

### 🚀 4️⃣ 자동 생성

터미널에서 명령 실행:

```bash
npm run generate-menu
```

**출력 예시:**
```
✅ 8개의 카테고리를 파싱했습니다.
✅ 8개의 매핑을 파싱했습니다.
   매핑: { '1': 'pho', '2': 'friedNoodles', ... }
✅ 57개의 메뉴 아이템을 파싱했습니다.

📊 카테고리별 메뉴 개수:
   칼국수 (pho): 5개
   볶음면 (friedNoodles): 2개
   ...
```

✨ `data/menu.ts` 파일이 자동으로 생성되고 웹사이트에 즉시 반영됩니다!


## 💡 카테고리 매핑 시스템

### 작동 방식

1. **매핑 정의**: `menu-database.md` 상단에 숫자와 카테고리 ID 매핑
2. **메뉴 입력**: 메뉴의 카테고리 필드에 숫자만 입력
3. **자동 변환**: 스크립트가 숫자를 실제 카테고리 ID로 변환

### 장점

✅ **간편함**: 긴 영문 ID 대신 1-8 숫자만 입력
✅ **가독성**: 숫자로 한눈에 카테고리 확인
✅ **오타 방지**: 숫자만 입력하므로 오타 가능성 감소
✅ **유연성**: 매핑 변경만으로 카테고리 전환 가능

### 예시

```markdown
# 매핑 정의
1: pho (칼국수)
6: hotfood (안주류)

# 메뉴 입력
### 자들양지 쌀국수
- **카테고리**: 1  👈 "pho"로 자동 변환

### 점보계란말이
- **카테고리**: 6  👈 "hotfood"로 자동 변환
```

## 📐 아키텍처

### 관심사의 분리

1. **카테고리 레이어** (`data/category-database.md`)
   - 카테고리 정의
   - 순서 관리
   - 설명 관리

2. **매핑 레이어** (`data/menu-database.md` 상단)
   - 숫자와 카테고리 ID 매핑
   - 1-8 숫자로 간편하게 사용

3. **메뉴 레이어** (`data/menu-database.md`)
   - 메뉴 정보만 포함
   - 카테고리를 숫자로 참조

4. **데이터 레이어** (`data/menu.ts`, `hooks/`)
   - 자동 생성된 TypeScript 파일
   - 비즈니스 로직 처리

5. **UI 레이어** (`components/ui/`)
   - 재사용 가능한 순수 UI 컴포넌트
   - 디자이너 작업 영역

6. **기능 레이어** (`components/features/`)
   - UI 컴포넌트 조합
   - 비즈니스 컴포넌트

7. **페이지 레이어** (`app/`)
   - 데이터와 UI 통합

### 역할별 작업 영역

- **카테고리 관리자**: `data/category-database.md`
- **메뉴 담당자**: `data/menu-database.md` (카테고리를 1-8 숫자로!)
- **디자이너**: `components/ui/` 스타일 수정
- **프론트엔드**: `components/features/`, `app/` 수정
- **백엔드**: `data/*.md` → API로 대체
